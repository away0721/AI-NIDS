# Day1

## 项目目标

实现一个基于 CNN-LSTM 的网络入侵检测系统（AI-NIDS）。

## 我的理解

传统 IDS：
根据规则判断攻击。

例如：

- 10秒访问10个端口
- SSH频繁连接22端口

AI IDS：

输入网络流量特征。

由模型判断：

- Normal
- PortScan
- DDoS
- BruteForce

等攻击类型。

## 今日核心理解

1. AI-NIDS 本质是一个分类问题。

2. 网络流量经过特征提取后会变成特征向量。

3. CNN 学习特征之间的模式。

4. LSTM 学习特征随时间变化的关系。

5. 模型最终输出攻击类型。

6. 数据集比模型更重要，数据决定上线，模型决定下线。

7. AI-NIDS 的输入是网络流量特征向量，输出是攻击类型。

8. 模型学习的是特征之间的关系，而不是直接学习 TCP、UDP 等协议。

## 今日进一步理解

1. Baseline（基线模型）是用于对比实验的参考模型。

2. Baseline 不是简单跑一次，而是完整训练、测试并评估结果。

3. 机器学习项目需要通过 Baseline 验证所提出方法是否具有优势。

4. 不应该先选择模型，再寻找数据；应该先分析数据，再选择合适模型。

5. 模型没有高低贵贱，只有是否适合当前数据。

6. 对于表格数据，Random Forest、XGBoost 等树模型往往表现非常优秀。

7. CNN-LSTM 是否适合网络入侵检测，需要结合数据集特点判断，而不是因为它更复杂就一定更好。

8. 科研的目标不是证明自己的模型最强，而是验证自己的假设是否成立。

9. Overfitting（过拟合）本质是记住样本，而不是学会规律。

10. Underfitting（欠拟合）说明模型还没有学会数据规律。

11. Epoch 并不是越大越好，训练过久可能导致过拟合。

12. Accuracy 高并不一定代表模型优秀，需要结合 Recall、Precision 和 F1 一起分析。

13. 在 IDS 场景中，Recall（发现攻击的能力）往往比 Accuracy 更重要。

14. 不同攻击的检测难度不同，Bot 等攻击可能比 DDoS 更难识别。

## 今日最重要的收获

先观察数据，再决定模型。

不要因为学了某个模型，就强行让数据适配模型。
