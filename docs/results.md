# Results Guide

This document explains the main result files generated by the AI-NIDS project.

The `results/` directory stores model metrics, classification reports, confusion matrices, comparison charts, training curves, sample predictions, class imbalance experiment results, and error pattern analysis results.

## 1. Results Directory Overview

The current result structure is organized as follows:

```txt
results/
├── logistic_regression/
├── random_forest/
├── mlp_torch/
├── cnn_torch/
├── model_comparison/
└── multiclass/
    ├── logistic_regression/
    ├── random_forest/
    ├── mlp_torch/
    ├── cnn_torch/
    ├── model_comparison/
    ├── mlp_weight_experiments/
    ├── cnn_weight_experiments/
    ├── deep_learning_weight_experiments/
    ├── cnn_error_patterns/
    ├── random_forest_error_patterns/
    ├── clipped_weighted_cnn_error_patterns/
    ├── error_pattern_comparison/
    ├── class_error_rate_comparison/
    └── error_analysis_summary/
```

The result files are mainly divided into four parts:

1. Binary classification results
2. Multiclass classification results
3. Class imbalance experiment results
4. Error pattern analysis results

The clipped weight strategy helps reduce the side effects of direct class weighting.

The main IDS trade-off is:

```txt
Reducing missed attacks usually increases false alarms.
```

## 15. Error Types

| Error Type         | Description                                           |
| ------------------ | ----------------------------------------------------- |
| `Attack -> BENIGN` | Attack traffic is missed as normal traffic            |
| `BENIGN -> Attack` | Normal traffic is falsely reported as attack traffic  |
| `Attack -> Attack` | Attack traffic is classified as the wrong attack type |

## 23. Files Not Stored in GitHub

Large local files are intentionally excluded from GitHub:

```txt
data/
models/
.venv/
```

These directories contain raw datasets, processed datasets, trained model artifacts, and local virtual environment files.

The repository focuses on:

```txt
source code
documentation
selected metrics
reports
charts
summary-level analysis results
```

## 24. Notes

Some detailed intermediate files may be ignored in future versions, especially large error detail files such as:

```txt
*_error_details.csv
```

These files are useful for local analysis, but they are not always necessary for GitHub presentation. Summary-level CSV files and visualization charts are usually enough for project review.
